将前端项目容器化部署，可以保证环境一致性、便于版本回滚与横向扩展。本文以一个 React/Vue 构建产物为例，介绍完整的 Docker 部署方案。

六、CI/CD 小贴士

  • 利用 Docker 层缓存：先 COPY package*.json 再 npm ci，依赖不变时跳过安装步骤。
  • 镜像标签策略：使用 git commit hash 作为标签，便于回滚到任意版本。
  • 推送与拉取：推送到镜像仓库后，服务器只需 docker pull && docker-compose up -d 即可完成更新。

总结

多阶段构建 + nginx 托管静态文件，是前端容器化的标准做法。配合 .dockerignore、自定义 nginx 配置和 docker-compose 编排，可以搭建出轻量、稳定、易于维护的部署流水线，让前端部署从此告别”在我电脑上是好的”。
